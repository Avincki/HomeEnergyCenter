1. Overview & prerequisites

HomeEnergyCenter is a self-hosted Python service (FastAPI + uvicorn) that minimises electricity cost on a Belgian dynamic-tariff contract by orchestrating SolarEdge, sonnenBatterie, and HomeWizard devices. It runs as a long-lived web service that:

  • Polls local network devices (inverter, battery, P1 meter, EV charger) over HTTP and Modbus-TCP.
  • Stores time series in SQLite and exposes a dashboard / API on port 8000 by default.
  • Pulls day-ahead prices from ENTSO-E and a solar forecast from Forecast.Solar.
Network requirement The host must be on the same LAN as the energy hardware (default subnet in this project: 192.168.129.0/24). A cloud or off-site host will not work — Modbus and the local-API endpoints are only reachable inside the home network.

Hardware comparison

BoxSuitabilityNotes
Raspberry Pi 5 (8 GB) Excellent Cheap, silent, 5 W idle. The recommended target — orchestration is light I/O, no GPU needed.
Raspberry Pi 4 (4 GB+) Fine Slower install (slow wheel builds), but runtime load is trivial.
NVIDIA DGX Spark Overkill but works Only worth it if you also plan to run local LLMs / forecasting models on the same box. Same setup as any Ubuntu ARM64 host.

Common prerequisites

  • Python 3.11 or newer.
  • An always-on wired Ethernet connection to your home LAN.
  • A Tailscale account (free) — sign up at https://tailscale.com.
  • Your existing config.yaml (or config.example.yaml as a starting point), with device IPs, ENTSO-E API key, and tariff settings.

2. Option A — Raspberry Pi recommended

2.1 Flash the OS

  1. Download Raspberry Pi Imager from https://www.raspberrypi.com/software/.
  2. Insert a 32 GB+ A2-class microSD card (or use an SSD over USB-3 for better longevity).
  3. Choose Raspberry Pi OS Lite (64-bit, Bookworm). The desktop edition is unnecessary — this runs headless.
  4. Click the gear icon to pre-configure:
    • Hostname: energycenter
    • Username/password (don't use pi/raspberry).
    • Enable SSH (key-based recommended).
    • Set Wi-Fi if you can't use Ethernet (Ethernet is preferred for stability).
    • Locale: Europe/Brussels.
  5. Write, eject, boot the Pi.

2.2 First boot — update & install dependencies

SSH in from your laptop:

ssh youruser@energycenter.local

Bring the system up to date and install build prerequisites:

sudo apt update && sudo apt upgrade -y
sudo apt install -y \
    python3 python3-venv python3-pip python3-dev \
    git build-essential libssl-dev libffi-dev \
    sqlite3 curl ca-certificates

Verify Python:

python3 --version
# expect: Python 3.11.x or newer
Pi tip Bookworm ships with Python 3.11. If you're on an older Bullseye image, upgrade — building Python 3.11 from source on a Pi takes ~40 minutes and is unnecessary.

Now jump to section 4 (Install the application).

3. Option B — NVIDIA DGX Spark

The DGX Spark ships with DGX OS (Ubuntu-based, ARM64, GB10 Grace Blackwell). For a non-GPU service like HomeEnergyCenter, you treat it as a regular Ubuntu host — no CUDA wrangling required.

3.1 Initial setup

  1. Complete NVIDIA's first-boot wizard (account, network, DGX Cloud login if you use it).
  2. Plug into wired Ethernet on the same LAN as your inverter/battery/meter.
  3. Open a terminal (or SSH in from your laptop).

3.2 Install OS dependencies

sudo apt update && sudo apt upgrade -y
sudo apt install -y \
    python3 python3-venv python3-pip python3-dev \
    git build-essential libssl-dev libffi-dev \
    sqlite3 curl ca-certificates

Confirm Python ≥ 3.11:

python3 --version
If you also plan to run local models Keep HomeEnergyCenter in its own venv and its own systemd unit, separate from any CUDA / PyTorch environments. There's no reason for the orchestrator to share a Python environment with model code — segregating them makes upgrades safer.

Continue with section 4.

4. Install the application

The steps below are identical on Pi and DGX Spark.

4.1 Create a service user (recommended)

sudo useradd --system --create-home --home-dir /opt/energycenter --shell /bin/bash energycenter
sudo -u energycenter -i

You're now in /opt/energycenter as the energycenter user.

4.2 Get the code

Either clone from your git host:

git clone <your-repo-url> HomeEnergyCenter
cd HomeEnergyCenter

Or, if you're moving from the Windows machine, copy the working tree over scp / rsync. Exclude .venv/, __pycache__/, *.db, and any local secrets you don't want shipped.

# From the Windows source machine, e.g. (run in WSL or Git Bash):
rsync -av --exclude='.venv' --exclude='__pycache__' --exclude='*.db' \
  ./HomeEnergyCenter/ energycenter@energycenter.local:/opt/energycenter/HomeEnergyCenter/

4.3 Create a virtualenv and install

cd /opt/energycenter/HomeEnergyCenter
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip wheel
pip install -e .
Pi build time On a Pi 4, pymodbus and aiohttp wheels usually exist for ARM64; if pip falls back to source, expect 10–15 minutes. Be patient — it's a one-off.

5. Configure

5.1 Create config.yaml

cp config.example.yaml config.yaml
nano config.yaml

At minimum, set:

  • Device IPs — your sonnenBatterie, P1 meter, car charger, small/large solar, SolarEdge inverter. Use the static LAN IPs reserved on your router.
  • ENTSO-E API token — the day-ahead pricing key (was migrated to web-api.tp.entsoe.eu in the recent refactor).
  • Web bind — leave as: 
    web:
  host: 0.0.0.0
  port: 8000
    so the dashboard listens on every interface (LAN and the Tailscale virtual interface).
  • Timezone — Europe/Brussels for tariff alignment.

5.2 Initialise the database

The project uses Alembic migrations:

cd /opt/energycenter/HomeEnergyCenter
source .venv/bin/activate
alembic upgrade head

This creates the SQLite DB and applies all migrations. The next launch will reuse it.

6. First run & smoke test

Still in the venv:

python main.py

Expected output: structured logs from device pollers, the price provider, and uvicorn announcing it's listening on 0.0.0.0:8000.

From any device on the same LAN, open:

http://<host-lan-ip>:8000/

You should see the dashboard. Once verified, hit Ctrl+C and proceed to make it a service.

How to find the host's LAN IP 
ip -4 addr show | grep inet

Look for the 192.168.129.x address on eth0 (or end0 on newer kernels).

7. Run as a service (systemd)

A systemd unit gives you auto-start at boot, auto-restart on crash, and structured logs in journalctl.

7.1 Create the unit file

sudo nano /etc/systemd/system/energycenter.service

Paste:

[Unit]
Description=HomeEnergyCenter (energy orchestrator)
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=energycenter
Group=energycenter
WorkingDirectory=/opt/energycenter/HomeEnergyCenter
Environment=EO_CONFIG=/opt/energycenter/HomeEnergyCenter/config.yaml
ExecStart=/opt/energycenter/HomeEnergyCenter/.venv/bin/python main.py
Restart=on-failure
RestartSec=5
# Light hardening — orchestrator only needs LAN access and its own data dir.
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/energycenter/HomeEnergyCenter

[Install]
WantedBy=multi-user.target

7.2 Enable and start it

sudo systemctl daemon-reload
sudo systemctl enable --now energycenter.service
sudo systemctl status energycenter.service

Tail logs:

journalctl -u energycenter.service -f

7.3 Useful management commands

ActionCommand
Stopsudo systemctl stop energycenter
Restart (after config change)sudo systemctl restart energycenter
Disable autostartsudo systemctl disable energycenter
Last 200 log linesjournalctl -u energycenter -n 200 --no-pager

7.4 Open the firewall (if active)

Pi OS Lite has no firewall by default. If you've enabled ufw or are on DGX OS with one running, allow port 8000 on the LAN and Tailscale interfaces:

# Allow from LAN
sudo ufw allow from 192.168.129.0/24 to any port 8000 proto tcp
# Allow from Tailscale (CGNAT range)
sudo ufw allow from 100.64.0.0/10 to any port 8000 proto tcp

8. Tailscale on the host

8.1 Install

On both Pi OS and DGX OS (Ubuntu-based), the official one-liner works:

curl -fsSL https://tailscale.com/install.sh | sh

8.2 Authenticate

sudo tailscale up

The command prints a URL — open it in any browser, log into your Tailscale account, and approve the device. The host now has a stable 100.x.y.z address inside your tailnet.

8.3 Make sure it's discoverable

tailscale status
tailscale ip -4

Note the 100.x.y.z address — that's the only address you need to remember.

8.4 MagicDNS already enabled — tailnet: tail7b791a.ts.net

MagicDNS is on for this tailnet, so every device joined to it can reach the host by name instead of by 100.x.y.z address. Two equivalent forms work:

  • Short name (works from inside the tailnet): http://energycenter:8000
  • Fully qualified: http://energycenter.tail7b791a.ts.net:8000

The fully qualified form is the safer one to bookmark — it never collides with mDNS / local DNS suffixes on phones or laptops that join other networks.

8.5 Optional: enable HTTPS

If your phone browser nags about HTTP, Tailscale offers free certs. With HTTPS turned on for the tailnet, run on the host:

sudo tailscale cert energycenter.tail7b791a.ts.net

Then either front uvicorn with a tiny Caddy/nginx terminator, or pass the cert paths into uvicorn via the FastAPI app config. (Skip this until you actually feel the friction — HTTP-only inside Tailscale is end-to-end encrypted by the WireGuard tunnel anyway.)

9. Tailscale on your phone

  1. Install Tailscale from the App Store (iOS) or Play Store (Android).
  2. Open it, log in with the same account you used on the host.
  3. Toggle the VPN switch on. iOS will ask for VPN permission — approve it.
  4. In the app's device list, you should see energycenter with status connected.

9.1 Open the dashboard

From any browser on the phone (cellular, public Wi-Fi, anywhere — the tunnel doesn't care):

# Recommended — fully qualified MagicDNS name on tailnet tail7b791a.ts.net:
http://energycenter.tail7b791a.ts.net:8000

# Short MagicDNS form (also works inside the tailnet):
http://energycenter:8000

# Or by raw Tailscale IP if DNS is misbehaving:
http://<100.x.y.z>:8000

9.2 Add to home screen

  • iOS Safari: tap the share icon → Add to Home Screen. The dashboard now opens like an app.
  • Android Chrome: menu → Install app / Add to Home screen.

9.3 Performance and battery

Tailscale on idle is essentially free — the WireGuard tunnel only uses bandwidth/CPU when traffic flows. Leave it on.

10. Troubleshooting

The service won't start

sudo systemctl status energycenter
journalctl -u energycenter -n 100 --no-pager
  • config error — re-check config.yaml against config.example.yaml.
  • ModuleNotFoundError — venv not used; check ExecStart path in the unit file.
  • Address already in use — another process owns port 8000. Find it with sudo ss -ltnp | grep 8000.

Dashboard works on LAN but not over Tailscale

  • Confirm web.host: 0.0.0.0 in config.yaml (binding only to 127.0.0.1 excludes the Tailscale interface).
  • Check the host shows up in tailscale status on your phone.
  • If you have ufw active, see section 7.4.

Database errors after a code update

cd /opt/energycenter/HomeEnergyCenter
source .venv/bin/activate
alembic upgrade head
sudo systemctl restart energycenter

Modbus / device polling fails

  • The host is not on the same VLAN/subnet as the inverter. Move it, or set up a route.
  • The device IP changed (DHCP). Reserve a static lease on your router for each energy device.

Pi runs hot or throttles

HomeEnergyCenter alone won't tax a Pi 5, but a hot enclosure will. Check:

vcgencmd measure_temp
vcgencmd get_throttled
# 0x0 = healthy
